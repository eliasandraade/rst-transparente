Propostas vs. Central de Demandas — Escolha arquitetural

O projeto já tem um módulo de Propostas. Como ele se relaciona com a nova Central de Demandas?

Diferença conceitual

📋 Proposta atual
  • Sugestão livre, anônima opcional
  • Sem protocolo
  • Sem categorias
  • Sem histórico de movimentações
  • Status simples: Pendente → Respondida
  • Resposta única da gestão
  • Dados mínimos (só texto)
🎯 Demanda (nova)
  • Solicitação formal, identificada
  • Protocolo RST-ANO-NNNNNN
  • 10 categorias
  • Histórico completo de movimentações
  • 5 status com fluxo definido
  • Atualizações múltiplas da gestão
  • Nome, unidade, e-mail, telefone

Três abordagens possíveis

A

Módulos separados — manter Propostas intacto

Proposta continua como canal de sugestão anônima/informal. Demandas é um módulo novo, independente, com schema próprio.

✅ Zero risco de quebrar o que funciona
✅ Conceitos separados e claros
✅ Fácil de evoluir independentemente
✅ Admin navigation separada
⚠️ Dois módulos similares no sistema
⚠️ Usuário pode não entender diferença
⚠️ Mais código para manter
B

Migrar Propostas → Demandas (substituição)

Deprecar o módulo de Propostas. Migrar todos os dados existentes para o novo schema de Demanda. Um único canal unificado.

✅ Um único canal, sem confusão
✅ Admin mais simples
✅ Sem duplicação de conceitos
⚠️ Risco de quebrar fluxos existentes
⚠️ Migração de dados necessária
⚠️ Propostas eram anônimas; Demandas são identificadas — incompatível
⚠️ Força dados que Proposta não tinha
C

⭐ Separados, mas com componentes reutilizados (Recomendado)

Propostas permanece como canal informal (sugestão livre). Demandas é canal formal com protocolo. Reutilizar padrões de componentes, mas schemas independentes no banco.

✅ Sem risco de regressão
✅ Conceitos distintos preservados
✅ Reutiliza padrões visuais/API já testados
✅ Morador entende a diferença de intenção
✅ Evolutivo: futuramente pode integrar
⚠️ Dois módulos (aceito como trade-off)
⚠️ Comunicar diferença para o morador
💡 Minha recomendação: Opção C

Propostas = sugestão casual, sem compromisso. Demandas = solicitação formal com rastreabilidade.
São intenções diferentes do morador. Mesclar geraria confusão e risco sem benefício real.
No admin, os dois ficam no menu mas separados. No público, os dois são acessíveis mas comunicados com linguagem diferente.
